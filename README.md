Этап 1: Минимальный прототип с конфигурацией
Цель: Создать минимальное CLI-приложение с настраиваемыми параметрами через INI-файлы.

Реализовано:

 Парсинг INI-конфигурационных файлов

 Поддержка всех требуемых параметров:

Имя анализируемого пакета

URL репозитория или путь к тестовому файлу

Режим работы (local/remote/test)

Имя выходного файла с изображением

Режим ASCII-дерева

Максимальная глубина анализа

Подстрока для фильтрации

 Вывод всех параметров в формате ключ-значение

 Комплексная обработка ошибок для всех параметров

Валидация входных данных

Технические особенности:

Модульная архитектура с разделением ответственности

Автоматическое определение кодировки конфигурационных файлов

Кастомная система исключений для разных типов ошибок

Поддержка значений по умолчанию для всех параметров

Этап 2: Получение данных о зависимостях
Цель: Реализовать основную логику получения данных о зависимостях для их дальнейшего анализа и визуализации.

Реализовано:

 Парсинг Maven POM-файлов без использования менеджеров пакетов

 Извлечение информации о прямых зависимостях

 Поддержка формата Maven пакетов (groupId:artifactId:version)

 Разрешение переменных версий (${property.name})

 Обработка специальных версий (LATEST, RELEASE)

Вывод прямых зависимостей на экран

 Обработка сетевых ошибок и ошибок парсинга

Технические особенности:

Прямое HTTP-взаимодействие с Maven репозиториями

Парсинг XML с поддержкой namespace и без

Кэширование для оптимизации повторных запросов

Грейсфул деградация при частичных сбоях

Поддержка различных кодировок POM-файлов

Этап 3: Основные операции над графом
Цель: Построить граф зависимостей (с учетом транзитивности) и выполнить основные операции над ним.

Реализовано:

 Алгоритм BFS с рекурсией для построения графа

 Учет максимальной глубины анализа

 Фильтрация пакетов по подстроке

 Обнаружение и обработка циклических зависимостей

 Режим тестирования с тестовыми репозиториями

 Визуализация графа в читаемом формате

 Статистика по графу (количество пакетов, зависимостей)

Технические особенности:

Рекурсивный обход в ширину с контролем глубины

Механизм обнаружения циклов через отслеживание пути

Гибкая система фильтрации зависимостей

Поддержка тестовых репозиториев с пакетами A-Z

Детальная статистика и визуализация результатов

Этап 4: Дополнительные операции
Цель: Выполнить дополнительные операции над графом зависимостей.

Реализовано:

 Поиск обратных зависимостей для заданного пакета

 Алгоритм обхода графа для нахождения всех путей зависимостей

 Визуализация обратных зависимостей

 Демонстрация функциональности на тестовом репозитории

 Расширенные классы графов с дополнительными операциями

Технические особенности:

Алгоритм DFS с backtracking для поиска путей

Визуализация полных цепочек зависимостей

Поддержка как реальных, так и тестовых репозиториев

Детальная статистика по обратным зависимостям

Интеграция с предыдущими этапами
